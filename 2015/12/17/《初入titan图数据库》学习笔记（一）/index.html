《初入titan图数据库》学习笔记（一）

初入titan图数据库

小忍最近是各种狼闲的不顺，目前状态为三不被管地带；把搁置已久的titan重新学习一把，年龄大了，距离老年痴呆越来越近了，3个月前的程序竟废了一天的时间运行成功，想想也是醉了，以此作为教训，言归正传

titan的安装使用【windows】

1、下载https://github.com/thinkaurelius/titan/wiki/Downloads 安装文件【这里用的是0.5.4】版本

2、解压文件后在conf文件下修改titan-cassandra-es.properties文件，此文件中设置了对搜索引擎（es），Cassandra数据存储的配置，分别为：

storage.backend=cassandra
storage.hostname=192.168.90.11    本地的Cassandra

index.search.backend=elasticsearch
index.search.hostname=192.168.90.11 本地es服务

设置完之后，titan中的数据会存入到Cassandra中，对于使用es的使用还未做深入的研究。

3、想要运行titan的程序需要使用到elasticsearch，Cassandra，下面是针对elasticsearch，Cassandra的安装使用(基于windows)

首先、下载 https://www.elastic.co/downloads/elasticsearch
使用的是1.7.2版本

第二、解压安装文件有一下几个包：
bin是运行脚本，config是设置文件，lib是依赖的包。
启动解压目录下的bin名称的文件夹，双击elasticsearch.bat文件，就可以启动elasticsearch；

第三、启动成功后,会在解压目录下增加2个文件件,data用于数据存储, logs用于日志记录,可以自己创建plugins目录中用于放置自己的插件。

此时可以在浏览器中输入: http://localhost:9200/

“ok” : true,
“status” : 200,
“name” : “Glob Herman”,
“version” : { “number” : “0.20.6”, “snapshot_build” : false }, “tagline” : “You Know, for Search” }

出现上面结果,表示成功启动！

4、Cassandra的安装(windows)
首先、在官网下载，需注意的一点是，由于我下载的是3.1版本，本机使用的jdk为7版本，导致启动失败，需要jdk8的支撑 所以，小忍捯饬了半天换成2.1.5版本的Cassandra。在虚拟机及本地都进行测试成功。

第二、修改apache-cassandra-2.1.5\bin\目录下的Cassandra.bat文件对jvm的设置,默认的

set JAVA_OPTS=-ea^
 -javaagent:"%CASSANDRA_HOME%\lib\jamm-0.3.0.jar"^
 -Xms2G^
 -Xmx2G^

修改为：

set JAVA_OPTS=-ea^
 -javaagent:"%CASSANDRA_HOME%\lib\jamm-0.3.0.jar"^
 -Xms512M^
 -Xmx512M^

第三、修改apache-cassandra-2.1.5\conf\cassandra.yaml文件下针对ip的设置，及集群的设置

 listen_address: 192.168.90.11
 rpc_address:192.168.90.11
- seeds: "192.168.90.11" 这里可以是多个使用逗号隔开

OK，启动bin目录下的cassandra.bat文件。使用nodetool.bat tatus 查看Cassandra是否启动成功。

查看Cassandra中的数据库中的数据送cql语句进行查询，可以进入bin目录下的cqlsh.bat（由于window对cqlsh支持需要安装一些其他的组件狼闲的，所以小忍最终还是运行到linux）可直接进入./cqlsh 执行cql的语法。由于刚刚把Cassandra给安装成功，小试牛刀了一下几个命令，

describe keyspaces  查看有哪些数据库

CREATE KEYSPACE IF NOT EXISTS myCas WITH REPLICATION = {'class': 'SimpleStrategy','replication_factor':1};创建myCas数据库


 use mycas; 使用mycas数据库

create table user(id int , user_name varchar,PRIMARY KEY (id)); 创建表

describe tables  显示哪些表

insert into user (id,user_name) values(1,'zhangsan')；插入数据


select * from user; 查询

第三、编码应用

介绍下几个重要的类：
TitanGraph ：titan图数据的实现接口，主要用来进行图数据中的一系列的操作，包括创建表结构，获取图数据的管理器，进行查询等。

TitanFactory： 产生TitanGraph的工厂，参数是titan-cassandra-es.properties地址路径。

TitanManagement： 图数据管理器，用来创建节点，索引，主键，边等图数据的表结构，使用commit（）进行提交。

TitanTransaction ： 可向事务中添加节点（Vertex）

小忍也是刚刚接触titan图数据库，了解尚浅，附上代码，有助于理解一下下吧。

public class TitanGrapFactory {

public static TitanGraph g = null;
public static String cassandra_es_path;
public static Configuration conf;
private static Logger logger = LoggerFactory
        .getLogger(TitanGrapFactory.class);
private static TitanTransaction tx;
static TitanGrapFactory titanGrapFactory;
TitanGrapFactory() {
    try {
        conf = new Configuration(ReadConfig.getconf("titan.properties"));
        cassandra_es_path = conf.getValue("cassandra_es_path");
        g = TitanFactory.open(cassandra_es_path);
    } catch (IOException e) {
        logger.error("抛出异常：", e);
    }

}


public static TitanGrapFactory getInstance(){
    if(titanGrapFactory==null)titanGrapFactory =new TitanGrapFactory();
    return titanGrapFactory;
}

public  void createTableStructure() {
    // 图数据的管理器。
    TitanManagement mgmt = g.getManagementSystem();
    try {
        // 指定一个节点 。
        final PropertyKey personname = mgmt.makePropertyKey("personName")
                .dataType(String.class).make();
        // 索引
        TitanGraphIndex namej = mgmt.buildIndex("personName", Vertex.class)
                .addKey(personname).unique().buildCompositeIndex();
        mgmt.setConsistency(namej, ConsistencyModifier.LOCK); // 复合索引

        final PropertyKey entername = mgmt
                .makePropertyKey("enterpriseName").dataType(String.class)
                .make();
        TitanGraphIndex namei = mgmt
                .buildIndex("enterpriseName", Vertex.class)
                .addKey(entername).unique().buildCompositeIndex();
//            ElasticSearchIndex nameii=(ElasticSearchIndex) mgmt.buildIndex("enterpriseName", Vertex.class)
//                    .addKey(entername).unique().buildCompositeIndex();
        mgmt.setConsistency(namei, ConsistencyModifier.LOCK);
        final PropertyKey regNumber = mgmt.makePropertyKey("regNumber")
                .dataType(String.class).make();
        TitanGraphIndex regNumberj = mgmt
                .buildIndex("regNumber", Vertex.class).addKey(regNumber)
                .unique().buildCompositeIndex();
        mgmt.setConsistency(regNumberj, ConsistencyModifier.LOCK);// lock事务


        EdgeLabel enter2person = mgmt.makeEdgeLabel("enterprise2person")
                .multiplicity(Multiplicity.MULTI).make(); // 双向的
        EdgeLabel person2enter = mgmt.makeEdgeLabel("person2enterprise")
                .multiplicity(Multiplicity.MULTI).make();

        mgmt.makeVertexLabel("enterprise").make();
        mgmt.makeVertexLabel("person").make();

        mgmt.commit(); // 提交创建表结构
    } catch (SchemaViolationException e) {
        logger.info("创建表结构异常{}", e);
    }

}

public  List<String> queryEnterprise(String enterpriseName) {
    List<String> nameList = new ArrayList<String>();
    beginTransaction();
    Vertex enterprise;
    Iterable iterable1 = tx.query().has("label", "enterprise")
            .has("enterpriseName", enterpriseName).vertices();
    Iterator<Vertex> ite = iterable1.iterator();
    if (ite.hasNext()) {
        enterprise = ite.next();

        VertexQuery tmp = enterprise.query().labels("enterprise2person")
                .direction(Direction.OUT);

        Iterator<Vertex> personVertex = tmp.vertices().iterator();

        while (personVertex.hasNext()) {
            Vertex personVertex1 = personVertex.next();

            TitanEdge titanEdge = null;
            Iterable<TitanEdge> edges = ((TitanVertexQuery) tmp).adjacent(
                    (TitanVertex) personVertex1).titanEdges();
            Iterator<TitanEdge> ite11 = edges.iterator();
            if (ite11.hasNext()) {
                titanEdge = ite11.next();
            }
            StringBuffer position = new StringBuffer();
            String posStr = "";
            if (titanEdge != null) {
                Set set = titanEdge.getProperty("position");
                if (set != null) {
                    for (Object object : set) {
                        position.append(object.toString() + "/");
                    }

                    posStr = position.substring(0, position.length() - 1);
                }
            }

            String personName1 = personVertex1.getProperty("personName");
            String json = "{name:'" + personName1 + "', position:'"
                    + posStr + "'}";
            nameList.add(json);
        }
    }

    commitTransaction();

    return nameList;
}

public  List<String> queryPerson(String personName) {
    List<String> nameList = new ArrayList<String>();
    beginTransaction();

    Iterable iterable = tx.query()
            .has("personName", Compare.EQUAL, personName).vertices();
    if (iterable != null) {
        Iterator<Vertex> ite = iterable.iterator();
        if (ite.hasNext()) {
            person = (CacheVertex) ite.next();
        }


        TitanEdge titanEdge = null;
        Iterable<TitanEdge> edges = person.getEdges();
        Iterator<TitanEdge> iterEdges = edges.iterator();
        while (iterEdges.hasNext()) {
            titanEdge = iterEdges.next();

            Set set = titanEdge.getProperty("position");

                for (Object object : set) {
                    position.append(object.toString() + "/");
                }
                posStr = (position.substring(0, position.length() - 1));

                    .getProperty("enterpriseName");
            String regNumber = otherVertex.getProperty("regNumber");

            String nodeType = otherVertex.getProperty("nodeType");

                    + "'}";
            System.out.println("企业名JSON:" + json);

        }

    }

    commitTransaction();

}
